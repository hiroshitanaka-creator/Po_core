## 1. プロジェクト概要

**Po_core** は「哲学駆動 AI」です。39 人の哲学者 AI ペルソナがテンソル計算
(Freedom Pressure / Semantic Delta / Blocked Tensor) と 3 層の倫理ゲート
(W_Ethics Gate) を通じて議論し、倫理的に責任ある回答を生成します。

- バージョン: **`0.3.0`**
- Python: **≥ 3.10**
- フォーマッタ: black 26.1.0 / isort 5.13.2

### 3-C. SafetyMode による選択人数

| SafetyMode | 選択人数 | 条件 |
|---|---|---|
| NORMAL | 39 | 通常 |
| WARN | 5 | freedom_pressure ≥ 0.7 |
| CRITICAL | 1（dummy のみ） | freedom_pressure ≥ 0.85 |

### 3-D. `propose()` のフロー（base.py）

```
1. legacy_context を構築（テンソル値 + intent を dict に詰める）
2. self.reason(prompt, legacy_context) → raw dict を取得
3. normalize_response(raw) → {reasoning, perspective, tension, ...} に正規化
4. voice_loader.get_voice(module_id) → VoiceRenderer or None
5. voice が存在する場合:
     voiced_text = voice.render(prompt, tension_level, tensor_snapshot)
     normalized["voiced_reasoning"] = voiced_text   ← extra に保存
     ※ normalized["reasoning"] は上書きしない（2026-02-19 fix）
6. Proposal を生成:
     content = normalized["reasoning"]  ← 元の分析テキストを保持
     extra["voiced_reasoning"] = voiced_text  ← 表示用のスタイル付きテキスト
     extra["philosopher"] = name
     extra["tension"] = tension
     extra["normalized_response"] = {...}
7. [proposal] を返す
```

**Voice が content を上書きしなくなった理由（2026-02-19 fix）:**

以前は `normalized["reasoning"] = voice.render(...)` で元のテキストを破棄していた。
`InteractionMatrix._compute_tension()` は `Proposal.content` の文字列に対して
「freedom/determinism」「individual/collective」等の対立キーワードの出現を検索する。
voice テンプレート文字列にはこれらのキーワードが含まれないため、
tension = 0 → `high_interference_pairs()` 空 → deliberation 第2ラウンドが
no-op（`n_revised == 0`）になるというバグがあった。

修正後: `content` = 哲学的分析テキスト（テンソル計算に使用）、
`extra["voiced_reasoning"]` = スタイル付きレトリック（表示に使用）。

## 4. `run_turn` パイプライン（10 ステップ）

エントリポイント: `po_core.run()` (`src/po_core/app/api.py`) または
`PoSelf.generate()` (`src/po_core/po_self.py`)

実装: `src/po_core/ensemble.py`

```
Step 1: MemoryRead
  deps.memory_read.snapshot(ctx) → MemorySnapshot
  Emit: "MemorySnapshotted"

Step 2: TensorCompute
  deps.tensors.compute(ctx, memory) → TensorSnapshot
  {freedom_pressure, semantic_delta, blocked_tensor}
  Emit: "TensorComputed"

Step 3: SafetyMode 推定
  infer_safety_mode(tensors, config) → NORMAL / WARN / CRITICAL

Step 4: Solar Will（意図生成）
  deps.solarwill.compute_intent(ctx, tensors, memory) → Intent
  Emit: "IntentGenerated"

Step 5: IntentionGate（W_Ethics 第1ゲート・FAIL-CLOSED）
  deps.gate.judge_intent(ctx, intent, tensors, memory) → SafetyVerdict
  Emit: "SafetyJudged:Intention"
  REJECT/REVISE → fallback + Emit "SafetyDegraded"
  build_explanation_from_verdict() → ExplanationChain

Step 6: PhilosopherSelect（編成）
  deps.registry.select(mode) → SelectionResult
  deps.registry.load(ids) → [philosopher, ...]
  Emit: "PhilosophersSelected" {mode, n, cost_total, covered_tags}

Step 7: PartyMachine（並列実行）
  ThreadPoolExecutor でタイムアウト付き並列 propose()
  各哲学者 → [Proposal]
  Emit: "PhilosopherResult" {name, latency_ms, timed_out, error}

Step 8: DeliberationEngine（多ラウンド議論・Phase 2）
  InteractionMatrix.from_proposals() → NxN 調和/緊張行列
  high_interference_pairs() → 対立ペアを特定
  対立する哲学者ペアが counterargument を受け取り re-propose()
  Emit: "DeliberationCompleted" {n_rounds, total_proposals}

Step 9: ParetoAggregate（多目的最適化）
  5次元: safety / freedom / explainability / brevity / coherence
  Pareto front を計算 → SafetyMode に応じてウィナー選択
  Emit: "ParetoFrontComputed", "ParetoWinnerSelected"

Step 10: ActionGate（W_Ethics 第2ゲート・FAIL-CLOSED）
  deps.gate.judge_action(ctx, intent, proposal, tensors, memory) → SafetyVerdict
  ALLOW → proposal 返却（degraded=False）
  REJECT → fallback（degraded=True）
  Emit: "DecisionEmitted"
  deps.memory_write.append(record)
```

## 6. 安全システム（W_Ethics Gate）

`src/po_core/safety/wethics_gate/`

### 3 層構造

```
[Step 5: IntentionGate] ← 哲学者実行前に intent を判定
  W1: PromptInjectionDetector（インジェクション/DAN/ロールプレイ迂回）
  W3: EnglishKeywordViolationDetector v0.2（有害キーワード + 依存偽装パターン）
  obfuscation normalization（エンコード難読化の正規化）
  → ALLOW / CONSTRAIN / REJECT

[Step 9〜10: ActionGate] ← 集約後の proposal を判定
  同じ Detector 群
  → ALLOW / REJECT / REVISE
  REJECT → fallback proposal（degraded=True）

[ExplanationChain] ← Phase 3 追加
  SafetyVerdict → ExplanationChain（監査トレイル）
  build_explanation_from_verdict()
  Emit: "ExplanationEmitted"
```

## 7. アグリゲータ（Pareto 集約）

`src/po_core/aggregator/`

5 次元の多目的最適化:

| 次元 | 計算方法 |
|---|---|
| safety | policy.score（ゲート評価）|
| freedom | base_freedom(action_type) × (1 − freedom_pressure) |
| explainability | rationale × author_reliability（trace から）|
| brevity | 文字数逆数 |
| coherence | consensus × (1 − conflict_penalty) |

Pareto front: O(n²)、n ≤ 39

ウィナー選択:

- NORMAL: 重み付き和（pareto_table.yaml で設定可）
- WARN: safety 最大
- CRITICAL: brevity 最大
